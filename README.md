# tpgk-max-bot

Бот для платформы [MAX](https://max.ru) — сайт-справочник для абитуриентов Томского промышленно-гуманитарного колледжа (ОГБПОУ ТПГК) прямо в мессенджере: специальности, приёмная комиссия, часто задаваемые вопросы и другие разделы, которые наполняют сотрудники колледжа.

## Команды

| Команда              | Кто может  | Что делает                                                              |
| --------------------- | ---------- | -------------------------------------------------------------------------- |
| `/menu`               | все        | Открывает главное меню заново (свежим сообщением внизу чата)                |
| `/admin`               | employee, superuser | Панель управления контентом: добавить/отредактировать/удалить страницу и вложения |
| `/superuser`, `/superuser <имя>` | superuser | Панель ролей и поиск пользователей; из неё же — «Структура разделов»      |
| `/reg`                 | все        | Регистрация в базе (см. «Роли» ниже)                                       |

## Возможности

- **Публичное меню** — разделы и подкаталоги любой вложенности, страницы с текстом, фото и документами, постраничная навигация (батчами по 5, с хлебными крошками). Кнопка **🔄 Обновить** на главном меню шлёт свежее меню новым сообщением внизу чата — удобно после скачивания файла, когда исходное меню осталось выше.
- **`/admin`** — сотрудники добавляют, редактируют и удаляют страницы и вложения внутри уже существующей структуры; списки тоже постранично, чтобы не упереться в лимиты при большом числе разделов. Кнопка **🚪 Выйти в обычное меню** явно переключает из режима управления обратно в публичный просмотр, чтобы не путать один с другим.
- **`/superuser` → «Структура разделов»** — суперпользователь создаёт, переименовывает и удаляет разделы/подкаталоги. Бот предупреждает, если действие скроет уже существующие страницы из публичного меню.
- **Роли** — `user` → `employee` → `superuser`, обе ступени назначаются через `/superuser` прямо в боте (в обе стороны). Первый когда-либо зарегистрировавшийся человек автоматически становится `superuser` — без ручных правок в БД.
- **`/reg`** — обычные посетители в базу не попадают вовсе; запись создаётся только для тех, кто явно выполнил `/reg` (нужно тем, кого затем назначают сотрудником).

## Стек

TypeScript, [@maxhub/max-bot-api](https://www.npmjs.com/package/@maxhub/max-bot-api), PostgreSQL (`pg`), без веб-фреймворка — чистая архитектура (`domain` → `application` → `infrastructure`).

## Установка (локально, для разработки)

```bash
npm install
cp .env.example .env
# заполнить BOT_TOKEN и DATABASE_URL в .env
npm run migrate
npm run dev
```

### Развёртывание на сервере

Пошаговые инструкции «с нуля» (установка Node.js/PostgreSQL, systemd/служба Windows, автозапуск после перезагрузки, устранение неполадок):

- [`DEPLOY-WINDOWS.md`](DEPLOY-WINDOWS.md) — Windows 10 / Windows Server
- [`DEPLOY-LINUX.md`](DEPLOY-LINUX.md) — Debian/Ubuntu (и другие дистрибутивы Linux)

### Переменные окружения

| Переменная     | Описание                                             |
| -------------- | ----------------------------------------------------- |
| `BOT_TOKEN`    | Токен бота, выданный платформой MAX                    |
| `DATABASE_URL` | Строка подключения к PostgreSQL. Саму базу создавать заранее не нужно — `npm run migrate` создаст её сама, если её ещё нет. |

Подробности по базе данных, схеме и назначению первого суперпользователя — в [`DB.md`](DB.md).

## Структура проекта

```
src/
  domain/            сущности и порты (интерфейсы репозиториев) — без зависимости от MAX или Postgres
  application/        use-cases — вся бизнес-логика
  infrastructure/
    db/                миграции, автосоздание БД, seed-контент, реализация репозиториев на pg
    max/                обработчики бота (MaxBotServer.ts), сборка клавиатур и текстов сообщений
```

## Скрипты

| Команда         | Действие                                    |
| --------------- | -------------------------------------------- |
| `npm run dev`    | Запуск в режиме разработки (`ts-node`)        |
| `npm run build`  | Компиляция в `dist/`                          |
| `npm start`      | Запуск собранной версии из `dist/`            |
| `npm run migrate`| Создать БД, если её нет, применить схему и (на пустой базе) засеять начальный контент |
| `npm run seed:test`| **Полностью пересоздаёт** структуру разделов тестовыми данными «для вида» (4 обязательных раздела + специальности с кодами, страницы-заглушки) — см. [`DB.md`](DB.md) |

`.certs/` — российские корневые сертификаты, нужны для TLS-соединения с API MAX через `NODE_EXTRA_CA_CERTS` (публичные сертификаты, не секрет).
