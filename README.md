# tpgk-max-bot

Бот для платформы [MAX](https://max.ru) — сайт-справочник для абитуриентов Томского промышленно-гуманитарного колледжа (ОГБПОУ ТПГК) прямо в мессенджере: специальности, приёмная комиссия, часто задаваемые вопросы и другие разделы, которые наполняют сотрудники колледжа.

## Возможности

- **Публичное меню** — разделы и подкаталоги любой вложенности, страницы с текстом, фото и документами, постраничная навигация (батчами по 5).
- **`/admin`** — сотрудники добавляют, редактируют и удаляют страницы и вложения внутри уже существующей структуры.
- **`/superuser` → «Структура разделов»** — суперпользователь создаёт, переименовывает и удаляет разделы/подкаталоги, назначает и снимает роль сотрудника.
- **`/reg`** — разовая регистрация в базе (нужна только тем, кого потом собираются назначить сотрудником; обычные посетители в базу не попадают).

## Стек

TypeScript, [@maxhub/max-bot-api](https://www.npmjs.com/package/@maxhub/max-bot-api), PostgreSQL (`pg`), без веб-фреймворка — чистая архитектура (`domain` → `application` → `infrastructure`).

## Установка

```bash
npm install
cp .env.example .env
# заполнить BOT_TOKEN и DATABASE_URL в .env
npm run migrate
npm run dev
```

### Переменные окружения

| Переменная     | Описание                                             |
| -------------- | ----------------------------------------------------- |
| `BOT_TOKEN`    | Токен бота, выданный платформой MAX                    |
| `DATABASE_URL` | Строка подключения к PostgreSQL. Саму базу создавать заранее не нужно — `npm run migrate` создаст её сама, если её ещё нет. |

Подробности по базе данных, схеме и назначению первого суперпользователя — в [`DB.md`](DB.md).

### Роли

`user` (по умолчанию) → `employee` (назначается через `/superuser`) → `superuser` (назначается вручную в БД: `UPDATE users SET role='superuser' WHERE id=...`, после того как человек выполнил `/reg`).

## Структура проекта

```
src/
  domain/            сущности и порты (интерфейсы репозиториев)
  application/        use-cases — вся бизнес-логика, без зависимости от MAX или Postgres
  infrastructure/
    db/                миграции, seed-контент, реализация репозиториев на pg
    max/                обработчики бота, сборка клавиатур и текстов сообщений
```

## Скрипты

| Команда         | Действие                                    |
| --------------- | -------------------------------------------- |
| `npm run dev`    | Запуск в режиме разработки (`ts-node`)        |
| `npm run build`  | Компиляция в `dist/`                          |
| `npm start`      | Запуск собранной версии из `dist/`            |
| `npm run migrate`| Применить схему БД (и засеять начальный контент, если БД пуста) |

`.certs/` — российские корневые сертификаты, нужны для TLS-соединения с API MAX через `NODE_EXTRA_CA_CERTS` (публичные сертификаты, не секрет).
